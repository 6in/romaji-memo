## Problem

日本語テキストを入力欄に貼り付けて、ギャル・武士などのスタイルで変換しても、
入力した日本語がそのまま `converted` に返ってきてスタイル変換が行われない。

## Root Cause

`prompts.rs` のシステムプロンプトに以下のルールが明記されている:

```
- 入力はローマ字（ヘボン式・訓令式混在可）または英単語の混在テキストです
```

AI はこのルールに従い「ローマ字入力を変換するエンジン」として動作するため、
日本語が入力されると何もすべきことがないと判断し、入力をそのまま返す。

## Solution

システムプロンプトを修正して、日本語入力の場合はスタイル変換を行うことを明示する。

**方針 A（シンプル）:** 基本ルールに日本語入力への対応を追記:

```
- 入力がすでに日本語の場合は、ローマ字変換は不要。スタイル指示に従って
  文体・口調を変換してください（例: ギャル語・武士言葉への変換）
```

**方針 B（入力タイプ検出）:** Rust 側または TS 側で入力がローマ字か日本語かを
判定し、プロンプトを動的に切り替える。ただし実装が複雑になるため方針 A を先に試すこと。

変更後は `osaka`/`gal`/`bushi`/`okama` 各スタイルで日本語入力を試して動作確認すること。
