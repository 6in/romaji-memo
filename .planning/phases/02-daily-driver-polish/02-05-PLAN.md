**状態:**
- `editingId: string | null` — 現在編集中のプロバイダー ID
- `editForm: Partial<ProviderConfig>` — 編集フォームの値
- `confirmDeleteId: string | null` — 削除確認中のプロバイダー ID
- `isAdding: boolean` — 新規追加モード

**プロバイダーリスト:**
- `useProviderConfig()` で全プロバイダー設定を取得
- `useSettingsStore` の `activeProviderId` でアクティブ判定

**各プロバイダーカード:**
```tsx
<div className="bg-card border border-border rounded-md p-3 mb-2">
  <div className="flex items-center gap-2">
    {/* アクティブインジケーター — UI-SPEC: 8px green dot */}
    {activeProviderId === provider.id && (
      <div className="w-2 h-2 rounded-full bg-primary flex-shrink-0" />
    )}
    <span className="text-sm font-medium flex-1">{provider.name}</span>
    <span className="text-[10px] text-muted-foreground">{provider.adapter}</span>
  </div>
  {/* 展開時の編集フォーム (editingId === provider.id) */}
</div>
```

**インライン編集フォーム (展開時):**
- Name: `<Input className="text-sm" value={editForm.name} />`
- Adapter: `<select>` with options "anthropic", "openai"
- Base URL: `<Input className="text-xs font-mono" placeholder="https://api.openai.com/v1" />`
- Model: `<Input className="text-sm" />`
- API Key: `<Input type="password" className="text-xs font-mono" placeholder="sk-..." />`
  - API キーのプレースホルダー: 既存エントリで api_key が `"<encrypted>"` の場合は `placeholder="(変更する場合のみ入力)"` と表示。空送信なら Keychain は更新しない。
- Enabled: checkbox

**接続テスト (PROV-06):**
```tsx
<button
  onClick={() => pingMutation.mutate({ baseUrl: editForm.baseUrl || '', apiKey: editForm.apiKey })}
  disabled={pingMutation.isPending}
  className="bg-secondary text-secondary-foreground text-xs px-3 py-1 rounded-md"
>
  {pingMutation.isPending ? (
    <><Loader2 size={12} className="animate-spin inline mr-1" />テスト中...</>
  ) : '接続テスト'}
</button>
```

結果バッジ:
- 成功: `<span className="text-xs bg-primary/20 text-primary px-2 py-0.5 rounded">接続成功</span>`
- 失敗: `<span className="text-xs bg-destructive/10 text-destructive px-2 py-0.5 rounded">接続失敗: {error}</span>`

**保存/キャンセル/削除:**
- 保存: `upsertMutation.mutate(editForm as ProviderConfig)` — api_key が空文字なら送信しない (既存キーを維持)

- 削除: インライン確認 (UI-SPEC) — `confirmDeleteId === provider.id` のとき「本当に削除しますか？ [削除] [キャンセル]」
