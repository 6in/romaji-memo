## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline chip gaps |
| sm | 8px | Compact row padding (history items, buffer items) |
| md | 16px | Default panel padding, form field spacing |
| lg | 24px | Section padding inside SettingsDialog tabs |
| xl | 32px | Dialog content padding, major section breaks |
| 2xl | 48px | Not used in Phase 2 |
| 3xl | 64px | Not used in Phase 2 |

Exceptions:
- TitleBar fixed height: 32px (8 * 4 — already established in Phase 1)
- Icon-only touch targets (TitleBar buttons): 24px × 24px (p-1 = 4px + 12px icon + 4px = minimum 20px; acceptable for desktop-only)
- HistoryDrawer toggle button: 36px tall (py-1.5 + text-xs — matches Phase 1 baseline)
- DraftBuffer item drag handle: 20px × 32px (visually compact; DnD handle, not primary CTA)

Source: Phase 1 codebase (`HistoryDrawer.tsx`, `TitleBar.tsx`, `Converter.tsx`) — matching existing patterns exactly.

## Typography

| Role | Size | Weight | Line Height | Tailwind Classes |
|------|------|--------|-------------|-----------------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 | `text-sm font-normal` |
| Label | 12px (text-xs) | 600 (semibold) | 1.4 | `font-semibold text-xs` |
| Heading (dialog/section) | 16px (text-base) | 600 (semibold) | 1.2 | `text-base font-semibold` |
| Monospace (API key, URL field) | 12px (text-xs font-mono) | 400 (regular) | 1.5 | `text-xs font-mono` |

Notes:
- Phase 1 uses `text-sm` (14px) as the dominant body size throughout — maintain in Phase 2.
- `text-xs` (12px) is the standard for metadata labels, timestamps, badges, and secondary text — maintain in Phase 2.
- Headings appear only in SettingsDialog tab titles and section headers; use `text-base font-semibold`.
- API key and base URL inputs use `font-mono` to visually distinguish sensitive/technical values.
- No display-size text (20px+) in Phase 2 — this is a compact utility app, not a marketing page.
- **Weight system: 2 weights only — regular (400) for Body/Monospace, semibold (600) for Label/Heading.**

Source: Phase 1 codebase patterns (`HistoryDrawer.tsx`, `Converter.tsx`, `TitleBar.tsx`).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background`: oklch(15% 0.05 160) dark / oklch(97% 0.02 160) light | App background, textarea background, main surface |
| Secondary (30%) | `--card` / `--muted`: oklch(18–25% 0.05 160) dark | Cards, SettingsDialog body, DraftBuffer panel, HistoryDrawer expanded area |
| Accent (10%) | `--primary`: oklch(40% 0.15 160) | Reserved for: pin icon (active state), connectivity test success badge, active provider indicator, search match highlight |
| Destructive | `--destructive`: oklch(55% 0.2 25) | Delete history item button, delete buffer item button, delete custom style button, delete provider button |

Accent reserved for (explicit list):
1. Pin icon fill when a history item is pinned (`text-primary fill-primary`)
2. Active provider indicator dot in provider list
3. Connectivity test result badge — "接続成功" state only
4. Search keyword match highlight in history results (use `bg-primary/20 text-primary` span)

Do NOT use accent on: general hover states, all interactive elements, style selector badges (use `bg-secondary` instead).

Light theme: Same token names; values invert via `[data-theme="light"]` block in `src/index.css`. No additional tokens needed.

Source: `src/index.css` (Phase 1 dark-green theme — fully established).

## Component Inventory

New components Phase 2 adds. Each maps to a requirement.

### HistoryDrawer (extended from Phase 1)

Existing component extended with:
- Search input at top of drawer (HIST-04): `<Input>` shadcn component, placeholder "キーワード検索...", `Search` icon left-side adornment (12px)
- Style filter chips row (HIST-05): horizontal scroll row of `<Badge>` chips for each style; "すべて" chip selected by default; active chip uses `bg-primary text-primary-foreground`; inactive uses `bg-secondary text-secondary-foreground`
- Pin toggle on each history item row (HIST-06): `<Pin>` icon (12px) right-aligned in item row; filled `text-primary fill-primary` when pinned; `text-muted-foreground` when unpinned; tap target: full right 24px of item row
- Delete button on each history item row (HIST-07): `<Trash2>` icon (12px); visible only on item hover; color `text-muted-foreground hover:text-destructive`; no confirmation dialog (undo not in scope)
- Empty search state: see Copywriting section

### DraftBuffer (new panel, BUFF-01 through BUFF-04)

- Panel location: collapsible section between Converter output and HistoryDrawer toggle; collapses exactly like HistoryDrawer (resize window on open/close)
- Panel height when open: 200px fixed (window adds 200px when opened)
- Toggle button: mirrors HistoryDrawer toggle style — `w-full flex items-center justify-center gap-1 py-1.5 text-xs text-muted-foreground hover:text-foreground hover:bg-muted/50 transition-colors`; icon: `<FileText>` (12px); label: "バッファ (N)"  where N = item count
- Item row: 36px tall, `px-3 py-2`, divided by `divide-y divide-border`
- Drag handle: `<GripVertical>` icon (14px), `text-muted-foreground`, cursor `grab`; left-side of each item
- Item text: `text-xs text-foreground`, `truncate`, occupies remaining width
- Delete button per item: `<X>` icon (12px), `text-muted-foreground hover:text-destructive`, right-aligned, visible on hover only
- "全コピー" button: full-width button at bottom of panel; `bg-secondary text-secondary-foreground hover:bg-muted text-xs py-1.5`; icon: `<Copy>` (12px) left of label; label: "全コピー (N件)"
- Empty state: see Copywriting section

### StyleManager (new, CONV-07)

- Location: accessible via Settings dialog, "スタイル" tab
- List of custom styles: same card-row pattern as provider list
- Add button: `<Plus>` icon + "スタイルを追加" label; `bg-primary text-primary-foreground text-xs px-3 py-1.5 rounded-md`
- Inline edit form per style: name field (`text-sm`) + prompt textarea (`text-xs font-mono`, 4 rows); save/cancel inline
- Delete button: `<Trash2>` icon; triggers inline confirmation (see Copywriting)

### SettingsDialog (new root, PROV-05/06/08, HIST-08)

- Trigger: gear icon `<Settings>` (14px) added to TitleBar, between theme toggle and minimize button
- Dialog width: 480px fixed (Tauri window is 420px wide — dialog rendered inside Tauri window using full-width sheet/drawer, not a floating dialog)
- Implementation: use shadcn `<Sheet>` (side="right", full-height) rather than `<Dialog>` to stay within narrow window constraints; width: 100% of window
- Tabs: `<Tabs>` component with 3 tabs — "プロバイダー" / "スタイル" / "全般"
- Tab height: 36px tab bar + content area fills remaining sheet height
- Close button: `<X>` (14px) top-right of sheet header

#### ProviderSettings tab

- Provider list: card rows, `bg-card border border-border rounded-md p-3 mb-2`
- Active provider indicator: filled green dot (8px circle, `bg-primary`) left of provider name
- Provider name: `text-sm font-medium`
- Edit inline: expand on click — shows name input, base_url input (font-mono), model input, API key input (font-mono, password masking), Save/Cancel buttons
- "接続テスト" button: `bg-secondary text-secondary-foreground text-xs px-3 py-1 rounded-md`; result displayed as inline badge — success: `bg-primary/20 text-primary` "接続成功" / failure: `bg-destructive/10 text-destructive` "接続失敗: {message}"
- Add provider button: same style as StyleManager add button

#### HistorySettings tab ("全般" tab)

- History limit input: labeled "履歴の保存件数" + number input (right-aligned, 80px wide); suffix label "件"; default 1000
- Save button: `bg-primary text-primary-foreground text-xs px-4 py-1.5 rounded-md`; label: "設定を保存"

## Interaction Contracts

### Global Hotkey (WINX-02)

- Shortcut: Cmd+Shift+R (macOS) / Ctrl+Shift+R (Windows)
- Behavior: toggles window visibility; if hidden → show + set_focus; if visible → hide
- No UI indicator needed — hotkey is system-level, invisible to in-app UI

### Search Input Debounce (HIST-04)

- Debounce: 300ms after last keystroke before invoking FTS5 search
- During debounce: no spinner, input remains active
- Zero results: show empty state (see Copywriting)

### DnD Reorder (BUFF-02)

- Library: @dnd-kit/sortable
- Visual feedback during drag: item lifts with `opacity-70 shadow-lg scale-[1.02]` via CSS transform
- Drop zone indicator: `border-t-2 border-primary` above target position
- Cursor: `grab` at rest, `grabbing` during drag

### Pin Toggle (HIST-06)

- Optimistic update: pin state flips immediately in UI, then syncs with DB
- Pinned items always sorted to top of history list (backend handles sort order)
- Visual: Pin icon fills solid (`fill-primary`) when pinned; outline when unpinned

### Delete Confirmation (HIST-07, BUFF-03, CONV-07 custom style delete, provider delete)

| Action | Confirmation Approach |
|--------|-----------------------|
| 履歴削除 (HIST-07) | No confirmation — delete immediately; undo not in scope |
| バッファ削除 (BUFF-03) | No confirmation — ephemeral data, delete immediately |
| カスタムスタイル削除 (CONV-07) | Inline confirmation: replace delete button with "本当に削除しますか？ [削除] [キャンセル]" in same row |
| プロバイダー削除 | Inline confirmation: same pattern as custom style delete |
