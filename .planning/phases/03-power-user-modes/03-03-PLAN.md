  </what-built>
  <how-to-verify>
    `npx tauri dev` でアプリを起動し、以下を順番に確認:

    **1. ミニモード (WINX-05)**
    1. TitleBar のミニモードボタン (縮小アイコン) をクリック
    2. ウィンドウが入力欄のみの高さ (~120px) に縮小されることを確認
    3. HistoryDrawer と DraftBuffer が非表示であることを確認
    4. もう一度ボタンをクリックして元のサイズに復帰することを確認
    5. 復帰後のサイズがミニモード前と同じであることを確認

    **2. クリップボード監視 (WINX-06)**
    1. TitleBar のクリップボード監視ボタンをクリック (アイコンがハイライトされる)
    2. 別のアプリ (テキストエディタ等) で適当なテキストをコピー (Cmd/Ctrl+C)
    3. アプリの入力欄にコピーしたテキストが自動的に入ることを確認
    4. アプリ内の「コピー」ボタンで変換結果をコピーしても入力欄が上書きされないことを確認
    5. 監視ボタンをもう一度クリックして監視を停止、以降は外部コピーが反映されないことを確認

    **3. 長文書モード (CONV-08)**
    1. TitleBar の長文書モードボタン (FileText アイコン) をクリック
    2. DocumentMode UI に切り替わることを確認
    3. テキストを入力して変換ボタンを押す
    4. 変換結果が蓄積リストに追加されることを確認
    5. 2-3 段落追加して蓄積リストに複数段落が表示されることを確認
    6. 段落の削除ボタンで個別削除できることを確認
    7. 「.md でエクスポート」をクリックし、OS の保存ダイアログが表示されることを確認
    8. 保存先を選んでファイルが正しく書き出されることを確認
    9. 長文書モードボタンをもう一度クリックして通常モードに戻ることを確認

    **4. 機能間の競合確認**
    1. ミニモード中に長文書モードボタンが無効化されていることを確認
    2. 長文書モード中にクリップボード監視を有効にし、外部コピーが入力欄に反映されることを確認
